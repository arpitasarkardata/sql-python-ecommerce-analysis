# 🏢 Enterprise Data Analysis & ERP Module Simulation

## 📌 Project Overview

A business was sitting on 100,000+ orders spread across 7 separate tables — with no visibility into revenue drivers, customer behaviour, or operational performance across its Sales, Inventory, and Customer modules.

This project simulates an **end-to-end ERP data analysis pipeline** — mirroring the structure of SAP SD (Sales & Distribution), MM (Materials Management), and CRM (Customer Relationship Management) modules — using Open SQL-compatible queries (MySQL/PostgreSQL), Python automation, and structured business reporting.

The output is a full suite of executive-ready deliverables: SQL-driven data extraction, Python visualisations, and documented business insights — directly replicating the kind of technical reporting and documentation work performed in SAP ABAP development environments.

## 🗂 Dataset — ERP Module Mapping

**Brazilian Ecommerce Dataset** — 7 interconnected tables, 100,000+ records

This dataset is structured to mirror a real ERP system. Each table maps to a functional SAP module:

| Table | Rows | ERP Module Equivalent | Description |
|---|---|---|---|
| customers | ~100K | SAP CRM — Customer Master | Customer IDs, cities, states |
| orders | ~100K | SAP SD — Sales Orders | Order timestamps, status, lifecycle |
| order_items | ~115K | SAP SD — Line Items / SAP MM — Inventory | Products, sellers, prices |
| products | ~33K | SAP MM — Material Master | Category, dimensions, attributes |
| payments | ~104K | SAP FI — Financial Accounting | Payment type, value, installments |
| sellers | ~3K | SAP SD — Partner Functions | Seller location and performance |
| geolocation | ~1M | SAP Basis — Regional Master Data | Zip codes, coordinates |

## 🔍 Analysis Structure — ERP Report Simulation

All SQL queries were written using Open SQL-compatible syntax (MySQL/PostgreSQL), following the same structured data extraction logic used in SAP ABAP report development.

### Module 1 — Sales & Revenue Reporting (SAP SD equivalent)
> Simulates revenue extraction queries a SAP ABAP developer would write for SD module reports

1. Total sales per product category — category-level revenue aggregation
2. Percentage of total revenue per product category — share-of-wallet analysis
3. Total revenue generated by each seller, ranked — seller performance report
4. Cumulative sales per month for each year — period-end financial rollup
5. Year-over-year growth rate of total sales — management KPI reporting

### Module 2 — Customer Analytics (SAP CRM equivalent)
> Simulates customer master data queries and retention reporting

6. Number of customers from each state — geographic distribution analysis
7. Moving average of order values per customer — behavioural trend analysis
8. Customer retention rate (6-month window) — CRM retention metric
9. Top 3 customers by spend per year — key account identification

### Module 3 — Operations & Order Management (SAP MM equivalent)
> Simulates operational data extraction and order processing reporting

10. Number of orders per month in 2018 — order volume trend analysis
11. Percentage of orders paid in installments — payment behaviour reporting

## 📈 Key Business Insights

- **Bed Table Bath** is the top revenue category — contributing **10.7%** of total revenue (SD module finding)
- **99.99%** of orders involve installment payments — critical input for FI module payment configuration
- Revenue grew **~20% year-over-year** from 2017 to 2018 — positive SD KPI trend
- **Customer retention is critically low** — majority of customers are one-time buyers, a CRM module risk flag
- **São Paulo** has the highest customer concentration — relevant for MM regional stock planning
- **Mid-year months (May–August)** consistently show highest order volumes — seasonal demand pattern for SD planning

## ▶️ How to Run

1. Clone this repository
```bash
git clone https://github.com/arpitasarkardata/sql-python-ecommerce-analysis.git
```

2. Install required Python libraries
```bash
pip install pandas mysql-connector-python matplotlib seaborn openpyxl
```

3. Set up MySQL and create the database
```sql
CREATE DATABASE ecommerce;
```

4. Update the database credentials in the notebook (Cell 1)
```python
host = "localhost"
user = "your_username"
password = "your_password"
database = "ecommerce"
```

5. Place the 7 CSV files in your local folder and update the `folder_path` variable

6. Run **Kernel → Restart & Run All** in Jupyter Notebook
