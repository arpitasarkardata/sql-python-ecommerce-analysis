# 🛒 Ecommerce Data Analysis — SQL + Python

## 📌 Project Overview

A business was sitting on 100,000+ orders spread across 7 separate tables — but had no clear visibility into what was driving revenue, who the best customers were, or whether the business was growing.

This project solves that problem by building an **end-to-end data analysis pipeline** using MySQL and Python — from raw CSV ingestion to executive-ready business insights.

## 🗂 Dataset

**Brazilian Ecommerce Dataset** — 7 interconnected tables, 100,000+ records

| Table | Rows | Description |
|---|---|---|
| customers | ~100K | Customer IDs, cities, states |
| orders | ~100K | Order timestamps, status |
| order_items | ~115K | Products, sellers, prices |
| products | ~33K | Category, dimensions |
| payments | ~104K | Payment type, value, installments |
| sellers | ~3K | Seller location |
| geolocation | ~1M | Zip codes, coordinates |

---

## 🔍 Analysis Structure

### Basic Queries
1. Find the total sales per product category
2. Count the number of customers from each state
3. Calculate the percentage of orders paid in installments

### Intermediate Queries
4. Calculate the number of orders per month in 2018
5. Calculate the percentage of total revenue per product category
6. Calculate the total revenue generated by each seller, ranked
7. Calculate the moving average of order values per customer

### Advanced Queries
8. Calculate cumulative sales per month for each year
9. Calculate the year-over-year growth rate of total sales
10. Calculate the customer retention rate (6-month window)
11. Identify the top 3 customers by spend per year

## 📈 Key Business Insights

- **Bed Table Bath** is the top revenue category — contributing **10.7%** of total revenue
- **99.99%** of orders are paid via installments — almost universal across customers
- Revenue grew **~20% year-over-year** from 2017 to 2018
- **Customer retention is very low** — the majority of customers are one-time buyers, a key business risk
- **São Paulo** has the highest customer concentration of any state
- **Mid-year months (May–August)** consistently show the highest order volumes

## ▶️ How to Run

1. Clone this repository
```bash
git clone https://github.com/YourUsername/ecommerce-sql-python-analysis.git
```

2. Install required Python libraries
```bash
pip install pandas mysql-connector-python matplotlib seaborn
```

3. Set up MySQL and create the database
```sql
CREATE DATABASE ecommerce;
```

4. Update the database credentials in the notebook (Cell 1)
```python
host = "localhost"
user = "your_username"
password = "your_password"
database = "ecommerce"
```

5. Place the 7 CSV files in your local folder and update the `folder_path` variable

6. Run **Kernel → Restart & Run All** in Jupyter Notebook
